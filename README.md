# Neurochemical Personality Simulation - Extreme Scale

## Project Overview

This project tests the hypothesis that **personality emerges as an adaptive construct** from the brain organizing biochemical reactions to maintain homeostasis, accumulate experience, and ensure reproduction.

**Key Principle:** We do NOT model personality directly. Instead, we test whether proto-personality can EMERGE from the interaction of computational components: neurochemistry, homeostasis, reinforcement learning, Theory of Mind, and meta-learning.

## Architecture

### Computational Modules

1. **Neurochemical System**
   - Dopamine, Serotonin, Cortisol, Norepinephrine, Acetylcholine
   - Realistic dynamics: receptor desensitization, temporal decay, cross-system interactions

2. **Homeostatic Module**
   - Monitors: safety, social connection, curiosity
   - Drives intrinsic motivation

3. **Hierarchical Reinforcement Learning (HRL)**
   - Meta-controller selects skills
   - Skills are specialized sub-policies
   - 32 learnable skills per agent

4. **Meta-Learning (RL²)**
   - GRU-based adaptation
   - Learns to learn across episodes

5. **Theory of Mind (ToM)**
   - Models other agents' states and intentions
   - Predicts actions, decides cooperation
   - Influenced by social status

6. **Prefrontal Cortex Planner**
   - GOAP (Goal-Oriented Action Planning)
   - Value-based action gating
   - 8 default personality rules

7. **World Model**
   - VAE encoder/decoder
   - GRU dynamics prediction

### Agent Types
- **Type 0 (~33 agents):** Hierarchical RL + Theory of Mind
- **Type 1 (~33 agents):** Meta-Learning only
- **Type 2 (~34 agents):** Baseline Actor-Critic


Reduce model size in `src/main.rs`:
```rust
const HIDDEN_DIM: i64 = 256;
const BATCH_SIZE: usize = 1024;

// In main():
let mut agents: Vec<PersonalityAgent> = (0..50)  // Reduce from 100
    .map(|i| { /* ... */ })
    .collect();
```

### Training Too Slow

1. **Verify GPU usage:**
   ```bash
   rocm-smi
   # Should show 90%+ GPU utilization
   ```

2. **Increase batch size** (better GPU utilization):
   ```rust
   const BATCH_SIZE: usize = 4096;
   ```

3. **Reduce logging** (less I/O overhead):
   ```rust
   const LOG_INTERVAL: usize = 500_000;
   ```

**Use faster storage for checkpoints:**

